# 쇼핑몰 AI Assistant
FastAPI와 LLM을 활용한 쇼핑몰 고객 문의 응답 API 테스트

가상의 종합 쇼핑몰 승승장구몰 : 창업 6년차, 회원 약 80만, 일 주문 1.2만 건, 상품 약 15만 SKU의 회사 

- 문제 배경
    - **CS 문의 폭증** — 배송·반품·교환·사이즈·재입고 문의가 하루 수천 건. 상담사가 못 따라갑니다.
    - **정보 과잉** — 상품 상세·리뷰가 너무 많아 고객이 핵심을 못 찾고, 다시 문의로 유입되는 악순환.
    - **문서 산재** — 사내 규정·정책·매뉴얼이 흩어져 있어 **직원조차** 못 찾습니다.
    - **글로벌화** — 해외 직구/셀러 상품이 늘며 상품명·설명의 번역·정규화가 필요합니다.
- 최종 산출물
    
    
    |  | 기능 | 한 줄 설명 |
    | --- | --- | --- |
    | 1 | **응용 서비스** | 리뷰 요약 · 상품 번역 · 상담 챗봇 |
    | 2 | **사내규정 RAG** | 직원용 규정·정책 QnA — *분류 없이 RAG만으로 완성* |
    | 3 | **고객 CS RAG** | 정책 문서를 근거로 고객 답변 — *먼저 문의 유형 분류로 라우팅* |
    | 4 | **문의 유형 분류기** | 고객 CS RAG의 라우터 — *회사 기준에 맞춘 파인튜닝 모델*** |
- 로드맵
    
    ```
    1. LLM 선택
       ↓  
    2. LLM API + 프롬프트로 동작하는 기능을 제작
       ↓ 
    3. 서비스화: 요약·번역·챗봇을 FastAPI로 서빙
       ↓  
    4. RAG: 우리 문서를 근거로 답하게 한다 (사내규정 RAG / 고객 CS RAG)
       ↓  
    5. 성능 평가: 수치로 증명 → '문의 분류가 회사 기준과 어긋남'을 발견
       ↓  
    6. 파인튜닝: 어긋나는 '분류'만 오픈소스 모델 + LoRA로 학습
       ↓  
    7. 통합: 모든 기능을 하나의 Assistant로 묶어 서빙·운영
    ```
    
- 개발 환경
    - **빠른 실험/학습 → Google Colab** : GPU도 빌려서 모델 학습에 사용.
    - **서비스/프로젝트 → 로컬 VS Code + miniconda(`genai`, Python 3.11)** : FastAPI 서버를 돌리고 배포를 준비
- 사용 기술
    
    
    | 개념 | 한 줄 정의 | 언제 쓰나 |
    | --- | --- | --- |
    | **RAG** (Retrieval-Augmented Generation) | 질문과 관련된 **문서를 검색해 프롬프트에 넣고** 그 근거로 답하게 하는 기법 | 회사 문서·정책처럼 **지식**을 근거로 답해야 할 때 |
    | **파인튜닝** (Fine-tuning) | 모델 가중치를 **우리 데이터로 추가 학습**해 행동을 바꾸는 기법 | 회사 기준에 맞춘 **일관된 분류/행동**이 필요할 때 |
